# תהליך העבודה – BluePin

תהליך 3-שלבי להמרת תכנית מסומנת ביד למסמך מקצועי.

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│   INPUT                  AI DRAFT             HUMAN REFINE       OUTPUT │
│  ─────────              ──────────           ──────────────     ──────  │
│                                                                         │
│   📐 תכנית               🤖 גרסה              👤 דיוק            📄 PDF  │
│   מקור (PDF)    ────►   ראשונית      ────►   ידני בכלי    ────► מקצועי  │
│                          (סקריפט)             ויזואלי                   │
│   ✏️ סימוני                                                              │
│   יד (PDF/JPG)                                                          │
│                                                                         │
│   📋 מקרא                                                                │
│   (DOCX/CSV)                                                            │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## שלב 1 – Ingest

קלט שהמשתמש מספק:
- **תכנית מקור** (PDF/DWG) – `התמצאות.pdf`
- **תכנית מסומנת ביד** (PDF/JPG) – `תכנית מסומנת WB.pdf`
- **מקרא** – `מקרא וולבוקס.docx`
- **הוראות** – `הוראות.pdf`

עיבוד אוטומטי:
```bash
# רינדור התכנית המקורית לתמונת רקע נקייה
python3 -c "
import fitz
mat = fitz.Matrix(300/72, 300/72)
fitz.open('התמצאות.pdf')[0].get_pixmap(matrix=mat).save('original.png')
"

# רינדור תכנית מסומנת + סיבוב כדי שטקסט יקרא
python3 -c "
import fitz
from PIL import Image
fitz.open('תכנית מסומנת WB.pdf')[0].get_pixmap(
    matrix=fitz.Matrix(300/72, 300/72)
).save('marked.png')
Image.open('marked.png').rotate(180, expand=True).save('marked_upright.png')
"

# חילוץ טקסט מהמקרא
unzip -o "מקרא וולבוקס.docx" -d /tmp/wb/
# קוראים word/document.xml כדי לחלץ ENTRIES
```

## שלב 2 – AI Draft

יצירת גרסה ראשונית עם נקודות במיקום משוער על סמך:
- תיאורי המקרא בעברית ("חצר משק", "מעבר שחקנים", ...)
- זיהוי ויזואלי של אזורים בתכנית המקורית
- מיפוי שמות לקטגוריות מקראיות

קוד: `generate_annotated.py` – מילון `WB_POSITIONS` עם 17 שמות וכ-20 נקודות.

תוצר ביניים: `marker-tool-standalone.html` מוכן עם הנקודות הראשוניות מוטמעות.

## שלב 3 – Human Refine (Interactive)

הלקוח פותח קובץ HTML יחיד (`סימון-וולבוקס.html`, 1.8MB):
1. רואה את תכניתו עם הנקודות שה-AI מיקם
2. גורר עיגול אדום/כחול למיקום הנכון
3. לוחץ על שם → תיבת שינוי שם
4. מחיקה/הוספה ישירה
5. רשימת נקודות בצד עם ניווט מהיר
6. שמירה אוטומטית ב-localStorage
7. כפתור "שמור JSON" → הורדת `markers.json`

טכנולוגיות: HTML/CSS/JavaScript נטו, ללא תלויות חיצוניות. תמונות מוטמעות כ-base64 → קובץ יחיד שניתן לשלוח בוואטסאפ/מייל. עובד בכל דפדפן מודרני, גם offline.

## שלב 4 – Final Render

הלקוח מחזיר את `markers.json`:
```bash
python3 render_from_json.py markers.json
```

תוצר סופי: `תכנית וולבוקס - חדשה.pdf` עם:
- תכנית מקורית כרקע
- וולבוקסים בעיגולים אדומים + שמות
- מצלמות בעיגולים כחולים
- מקרא מקצועי בצד ימין (RTL, פונט עברי)

## ארכיטקטורת קוד

```
┌──────────────────────────────────────────────────────┐
│ generate_annotated.py    →  AI Draft (מהזיכרון)      │
│                                                       │
│ marker-tool-template.html →  תבנית עם placeholders   │

│                                                       │

└──────────────────────────────────────────────────────┘
```

## נקודות עיצוב מרכזיות

1. **Single-file delivery** – הקובץ ללקוח חייב להיות יחיד. תלות בקבצים נפרדים = שבירה.
2. **No-backend** – הכל בדפדפן. אין הרשמה, אין שרת, אין עלות תפעולית.
3. **localStorage auto-save** – אם הלקוח סוגר בטעות → השינויים נשמרים.
4. **AI draft, not AI final** – לעולם לא נסגור את הלולאה בלי אישור אנושי. דיוק > אוטומציה.
5. **RTL/Hebrew first-class** – `python-bidi` ב-Python, dir="rtl" ב-HTML. גופן `Arial Unicode` שתומך בכל הסקריפטים.

## הרחבות עתידיות

- **OCR לקריאת שמות בכתב יד** – להפחית עוד את עבודת הדיוק.
- **Vision LLM לזיהוי עיגולים כחולים** – מיקום ראשוני אוטומטי מ-PDF.
- **תמיכה במספר קומות / רב-עמודים** – לחבילת PDF גדולה.
- **שיתוף לייב** – שניים על אותו לוח בו-זמנית (WebRTC P2P).
- **ייצוא ל-DWG/SVG** – עבור עורכים בתוכנות CAD.
- **תבניות לתעשיות נוספות** – HVAC, חשמל, פייפינג.
