# הוראות ביצוע - תכנית וולבוקס מסומנת

תיעוד התהליך ליצירת `תכנית וולבוקס - חדשה.pdf` עם סימוני וולבוקסים (אדום) ומצלמות (כחול).

## קבצי המקור

| קובץ | תיאור |
|------|-------|
| `התמצאות.pdf` | התכנית האדריכלית המקורית (landscape) – משמשת כבסיס |
| `תכנית מסומנת WB.pdf` | תכנית עם סימוני יד כחולים – הפניה למיקומי הוולבוקסים |
| `מקרא וולבוקס.docx` | מקרא שמות הוולבוקסים והכמויות (צד ימין של הפלט) |
| `הוראות.pdf` | הוראות המשתמש (וולבוקס באדום, מצלמה בכחול, מקרא בצד ימין) |
| `קומה 0 בורות.pdf` | תכנית קומה 0 – לעזר/הצלבה |

## דרישות התוצר

1. **בסיס** – התכנית המקורית, נקייה מסימוני יד.
2. **מיקומי וולבוקס** – עיגול אדום עם שם ה-WB.
3. **מיקומי מצלמות** – עיגול כחול.
4. **מקרא בצד ימין** – תואם לתוכן `מקרא וולבוקס.docx`.
5. עיגול כחול מודגש בתכנית המסומנת = נקודת וולבוקס (יוצג באדום בתוצר).

## שני מסלולי עבודה

### מסלול A – הגדרה ידנית קצרה (אוטומטי בקוד)

`generate_annotated.py` – סקריפט פייתון עם מילון מיקומי WB מוטמע. מתאים כשהמיקומים ידועים מראש.

```bash
python3 generate_annotated.py
```

לעדכון מיקומים: לערוך את הרשימה `WB_POSITIONS` בסקריפט (קואורדינטות פיקסל בתוך תמונה ברוחב 3508).

### מסלול B – סימון מדויק על ידי המשתמש (מומלץ)

כלי ויזואלי לסימון מדויק. שתי גרסאות:

- **`marker-tool.html`** – משתמש בקבצי `original.png` + `marked_upright.png` באותה תיקייה. שימוש מקומי.
- **`marker-tool-standalone.html`** – קובץ יחיד (1.8MB) עם התמונות מוטמעות. אפשר לשלוח בוואטסאפ/מייל לקפלן והוא יפתח אותו בדפדפן מכל מחשב.

1. פתיחת `marker-tool.html` (או `marker-tool-standalone.html`) בדפדפן (Chrome/Safari).
2. בצד שמאל – התכנית המקורית (לחיצה למיקום נקודה).
3. בצד ימין – התכנית המסומנת ביד (להתייחסות).
4. בחירת מצב למעלה: "וולבוקס (אדום)" / "מצלמה (כחול)".
5. בחירת שם מקליק מהיר (`WB MAIN A`, `WB B` וכו') או הקלדה ידנית.
6. לחיצה על המיקום בתכנית – הנקודה מתווספת.
7. גרירת נקודה כדי לכוונן, או לחיצה על ×‎ למחיקה.
8. כפתור **ייצא JSON** – שומר את הסימונים לקובץ `markers.json`.
9. כפתור **הצג כקובץ סופי** – מציג תצוגה מקדימה (קליק ימני → שמור תמונה).

ליצירת PDF סופי איכותי מ-JSON:
```bash
python3 render_from_json.py markers.json
```

## כלים שהיו בשימוש

- Python 3 + `PyMuPDF` (fitz) – המרת PDF לתמונה ברזולוציה גבוהה.
- `Pillow` (PIL) – ציור עיגולים, קווים וטקסט.
- `python-bidi` – עיבוד דו-כיווני לטקסט עברי לפני ציור (PIL לא מטפל ב-RTL אוטומטית).
- גופן `Arial Unicode.ttf` – תומך עברית + לטינית.
- HTML/JavaScript Vanilla (ללא תלויות חיצוניות) – כלי הסימון.

התקנת תלויות פייתון:
```bash
pip3 install PyMuPDF python-bidi pillow
```

## תהליך מהתחלה

```bash
# 1. רינדור התכנית המקורית
python3 -c "
import fitz
doc = fitz.open('התמצאות.pdf')
mat = fitz.Matrix(300/72, 300/72)
doc[0].get_pixmap(matrix=mat).save('original.png')
"

# 2. רינדור התכנית המסומנת (לעזר ויזואלי בכלי)
python3 -c "
import fitz
from PIL import Image
doc = fitz.open('תכנית מסומנת WB.pdf')
mat = fitz.Matrix(300/72, 300/72)
pix = doc[0].get_pixmap(matrix=mat)
pix.save('marked.png')
# סיבוב 180° כך שהטקסט בכתב יד יקרא בכיוון נכון
Image.open('marked.png').rotate(180, expand=True).save('marked_upright.png')
"

# 3. פתיחת הכלי וסימון
open marker-tool.html

# 4. לאחר ייצוא markers.json
python3 render_from_json.py markers.json
```

## איטרציה / שיפור

- **שינוי מיקום נקודה** – לפתוח את הכלי, לגרור את הנקודה, לייצא JSON מחדש, להריץ `render_from_json.py`.
- **תוכן המקרא** – לערוך את הרשימה `ENTRIES` בקבצים `generate_annotated.py` ו-`render_from_json.py`.
- **צבעים/גודל גופן** – הקבועים `RED`, `CAM`, ו-`font()` בראש הסקריפטים.
