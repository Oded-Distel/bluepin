# עיצוב: תכולת Wallbox

תאריך: 2026-06-09
קובץ יעד: `marker-tool.html`
מקור הדרישות: `from kaplan/הערות לסימון.md`, סעיף 4

## מטרה

כל Wallbox צריך רשימת תכולה פנימית (רשימת רכיבים עם כמויות) שניתן לערוך מלאה: הוספה, עדכון, ומחיקה. הרשימה צריכה להיות נגישה ויזואלית מהמרקר עצמו, ולהישמר עם שאר מצב הפרויקט.

## דרישות מהספק

- פריטים דיפולטיביים: Camera, BNC, XLR, Lemo, Coax, Network/Ethernet, Fiber LC, Fiber SC, Power Outlet
- פריטים מותאמים אישית (שם חופשי)
- כמות לכל פריט
- הוספה / עריכה / מחיקה בכל זמן

## החלטות עיצוב

### 1. פתיחת העורך
**קליק על העיגול האדום של ה-Wallbox → פאנל צד מימין מחליק לתוך המסך.**

- היום קליק על marker במצב default לא עושה כלום, אז אין רגרסיה.
- במצב כבל, ימשיך להפעיל `handleCableClick` (לוגיקה קיימת).
- מצלמות (`type === 'cam'`) לא מקבלות פאנל — תכולה היא WB-only לפי הספק.

### 2. הפאנל
- `position: fixed`, צמוד לקצה ימין, רוחב 320px, גובה 100vh.
- `z-index: 100` — מעל הפאנל הימני הקיים (`legend-side`) ומעל המרקרים, לא משנה את ה-layout.
- מבנה פנימי:
  - **כותרת**: שדה שם ה-WB (editable inline), כפתור X לסגירה
  - **גוף**: רשימת פריטים (`contents`) — `overflow-y: auto`, נגלל אם ארוך
  - **footer**: combobox "+ הוסף פריט" וכפתור "מחק Wallbox"

### 3. רשימת פריטים
כל שורה:
```
[שם פריט]  [−] [מספר] [+]  [×]
```
- שדה השם editable inline (לחיצה → קלט).
- Stepper כמות: `−` ו-`+` משנים ב-1; מינימום 1 (אם רוצים 0, מוחקים את הפריט).
- שדה מספר ניתן להקלדה ישירה (input type="number", min=1).
- `×` בצד שמאל מוחק את השורה.

### 4. Combobox הוספה
שדה אחד בתחתית הפאנל:
- placeholder: "+ הוסף פריט (BNC, XLR, ...)"
- בעת focus → רשימה מציעה את 9 הדיפולטים שאינם כבר ברשימה.
- הקלדה → רשימה מסוננת לפי טקסט (case-insensitive, מתחיל-עם).
- בחירה מהרשימה → מוסיף עם qty=1, סוגר את ההצעות, מנקה את השדה.
- Enter עם שם שלא ברשימה → מוסיף כפריט מותאם אישית עם qty=1.
- ESC → סוגר הצעות, מנקה שדה.
- **כפילויות**: מותרות. אם המשתמש מוסיף "BNC" כשכבר יש "BNC" ברשימה, נוצרת שורה חדשה. ניקוי כפילויות באחריות המשתמש (KISS, חוסך לוגיקה ערוכה).

### 5. תצוגה על המרקר (Badge)
- עיגול קטן 16x16px בפינה ימין-עליון של עיגול ה-WB, רקע כהה, טקסט לבן.
- מכיל את **סכום הכמויות** של כל הפריטים (לא מספר השורות).
- מוצג רק אם `contents.length > 0` ו-`sum(qty) > 0`.
- Tooltip על hover: רשימת פריטים מפורטת ("BNC ×4, XLR ×2").

### 6. שינוי שם
- שדה השם בכותרת הפאנל הוא **המקור היחיד** לעריכת שם WB.
- הקלקה על שם ה-WB ברשימה הימנית הקיימת (שהיום פותחת `prompt`) — תפתח כעת את הפאנל במקום זאת.
- Double-click על העיגול → פותח את הפאנל עם פוקוס מיידי על שדה השם (טוב לשינוי שם מהיר).
- ה-`prompt()` הקיים בשורה 610 נמחק.

### 7. סגירה
הפאנל נסגר ב:
- כפתור X
- ESC (כשאין פוקוס בשדה טקסט פתוח)
- קליק על WB אחר → מחליף תוכן, לא סוגר
- קליק על מצלמה / כבל / רקע → סוגר

### 8. שמירה ופרסיסטנס
- מבנה: `markers[i].contents = [{name: string, qty: number}, ...]`
- WB בלי שדה `contents` נחשב לרשימה ריקה (תאימות לאחור).
- `save()` הקיים שומר ל-localStorage — לא דורש שינוי, כי `contents` הוא חלק מ-`markers`.
- ייצוא JSON ("שמור JSON") יכלול אוטומטית את `contents`.

## ארכיטקטורת קוד (פיתוח ב-`marker-tool.html`)

קוד חדש שיתווסף:
- `DEFAULT_CONTENT_ITEMS = ['Camera', 'BNC', 'XLR', 'Lemo', 'Coax', 'Network', 'Fiber LC', 'Fiber SC', 'Power Outlet']` (ראש הקובץ)
- HTML element: `<aside id="contentsPanel" hidden>` (אחרי `main`)
- CSS חדש לפאנל: `.contents-panel`, `.contents-list`, `.contents-row`, `.qty-stepper`, `.combobox`, `.combobox-options`
- CSS חדש ל-badge: `.marker.wb .qty-badge`
- פונקציות חדשות:
  - `openContentsPanel(markerId)` — מציג את הפאנל למרקר נתון
  - `closeContentsPanel()` — מסתיר ומאפס state
  - `renderContentsPanel()` — מרנדר את התוכן מחדש מהמרקר הנוכחי
  - `addContentItem(name, qty=1)` — מוסיף שורה
  - `updateContentItem(idx, patch)` — מעדכן (שם/כמות)
  - `removeContentItem(idx)` — מסיר
  - `comboboxFilter(text)` — מסנן את הצעות הדיפולטים
- שינוי קל ב-`renderMarkers`: הוספת click handler לפתיחת הפאנל ל-WB; הוספת ה-badge.

state חדש: `let openContentsWbId = null;` — איזה WB פתוח כרגע בפאנל.

## מה לא בסקופ של העיצוב הזה

- **תצוגת תכולה ב-PDF הסופי** (`render_from_json.py`): נדחה למשימה נפרדת ב-roadmap (פיצ'ר #5).
- **ניהול תבניות / presets** (preset קיבוצי לתכולת WB): לא דרישה בספק, YAGNI.
- **חיפוש בכל הפריטים מכל ה-WB-ים**: YAGNI.

## סיכון ידוע

הפאנל הצף יסתיר חלק מהפאנל הימני הקיים (`legend-side`) כשהוא פתוח. זה מקובל כי המשתמש בעריכת WB ספציפי באותו רגע, אבל יש לוודא שאפשר לסגור מהר וחזרה לרשימה.

## בדיקות ידניות (Verification)

1. הוסף WB חדש → קליק עליו → פאנל נפתח, ריק, עם שדה שם.
2. הקלד "BNC" ב-combobox → הצעה מופיעה → לחיצה → מתווסף עם qty=1.
3. לחיצה על `+` → qty=2. badge על המרקר מציג "2".
4. הוספת פריט בשם "אחר" שלא ברשימה → Enter → מתווסף כמותאם אישית.
5. עריכת שם פריט inline → נשמר אחרי blur.
6. סגירה → רענון הדף → קליק על אותו WB → התכולה נשמרה.
7. שמירת JSON → קובץ כולל את שדה `contents`.
8. WB ישן בלי `contents` נטען מ-localStorage → לא קורס, מתפקד כריק.
